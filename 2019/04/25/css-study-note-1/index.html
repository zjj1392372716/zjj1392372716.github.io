《CSS世界》 阅读笔记

发布 : 2019-04-25 分类 : CSS 浏览 :

读了张鑫旭老师的《css世界》一书后，感触颇深「 非常棒 」。总结一些书中的重点知识，方便以后的查阅啦～～

CSS阅读笔记

一、CSS专用术语

1、属性

比如设置的 height 和 color 就是属性

2、值

值的分类比较广泛，大致我们可以分为：

  • 整数：z-index：1
  • 数值：line-height: 1.5;
  • 百分比：50%
  • 长度： 10px
  • 颜色： #fff

此外CSS3中还新增了时间值、角度值等。

3、关键字

如solid、transparent、inhert等都是关键字。

4、变量

css中的变量相对较少，CSS3 中的 currentColor。

5、长度单位

长度单位可以分为相对长度单位和绝对长度单位。

相对长度单位又可以分为是相对字体长度单位(em, rem, ex, ch)和相对视区长度单位(vh, vw, vmin, vmax)。
绝对长度单位最常见的和最常用的就是px了。

6、功能符

值以函数的形式指定.

比如： calc、url、attr、rgba、scale等等。

7、属性值

属性冒号后面的所有内容统一称为属性值。
例如，1px solid rgb(0,0,0)就可以称为 属性值，它是由“值+关键字+功能符”构成的。

8、声明

属性名加上属性值就是声明

9、声明块

声明块是花括号({})包裹的一系列声明

10、规则、规则集

出现了选择器，而且后面还跟着声明块。

11、选择器

  • 类选择器
  • id选择器
  • 属性选择器 比如： [title]{}、[title= "css-world"]{}、[title~="css-world"]{} 、 [title^= "css-world"]{ 和 [title$="css-world"]{}等。
  • 伪类选择器 一般指前面有个英文冒号(:)的选择器 比如 :first-child 或:last-child 等。
  • 伪元素选择器 就是有连续两个冒号的选择器，如::first-line::first-letter、::before 和::after。

12、关系选择器

  • 后代选择器 选择所有合乎规则的后代元素
  • 子选择器 > 仅仅选择合乎规则的儿子元素，孙子、重孙元素忽略
  • 兄弟选择器 ~ 选择当前元素后面的所有合乎规则的兄弟元素
  • 相邻兄弟选择器 仅仅选择当前元素相邻的那个合乎规则的兄弟元素

二、流、元素

何为流？

我们通常都说文档流，我们这里的流其实就是所说的文档流。文档流指的是元素排版布局过程中，元素会默认自动从左往右，从上往下的流式排列方式。

文档是页面所展现的内容比如div元素、p元素或者span元素等，而流则是展现内容的规则。

规则简单介绍如下：

通过这些规则，文档上的元素就会一定的方式展现出来。

1、流体布局

所谓“流体布局”，指的是利用元素“流”的特性实现的各类布局效果。因为“流”本身 具有自适应特性，所以“流体布局”往往都是具有自适应性的。但是，“流体布局”并不等同于 “自适应布局”。“自适应布局”是对凡是具有自适应特性的一类布局的统称，“流体布局”要狭 窄得多。例如，表格布局也可以设置为 100%自适应，但表格和“流”不是一路的，并不属于“流体布局”。

<div>是典型的具有“流”特性的元素，因此，曾经风靡的“div+CSS 布局”，实际上指的就是这里的“流体 布局”。

2、块级元素和内联元素

HTML 常见的标签有<div>、<p>、<li>和<table>以及<span>、<img>、<strong> 和<em>等。虽然标签种类繁多，但通常我们就把它们分为两类:块级元素(block-level element) 和内联元素(inline element)。

考点一：“块级元素”和“display 为 block 的元素”不是一个概念

考点二：块级元素与清除浮动的关系

在 block、table、list-item这几个的选择中，我们最好还是选择前两者，list-item并不好，一是因为这是li默认的display属性，前面会默认出现一个点，虽然我们可以设置list-style:none;清除。二是因为list-item在伪元素:before /:after中使用的话会在IE浏览器中出现一定的问题，因为IE11以下无法显示效果。

考点三：为什么List-item元素会出现项目点符号？

之所以 list-item 元素会出现项目符号是因为生成 了一个附加的盒子，学名“标记盒子”(marker box)，专门用来放圆点、数字这些项目符号。IE 浏览器下伪元素不支持 list-item 或许就是无法创建这个“标记盒子”导致的。

考点四：display:inline-block 盒子的出现

像这种存在嵌套关系的盒子我们称为是容器盒子。比如：inline-block外部是inline类型的盒子，内部是block类型的盒子。

现在，大家应该明白为何 display 属性值是 inline-block 的元素既能和图文一行显示， 又能直接设置 width/height 了吧!因为有两个盒子，外面的盒子是 inline 级别，里面的盒 子是 block 级别。

实际上，如果遵循这种理解，display:block 应该脑补成 display:block-block，display:table 应该脑补成 display:block-table，我们平时的写法实际上是一种简写。

width作用的细节

width 是作用在“内在盒子”上的，乍一看是一个普通的盒子，实际上， 这个“内在盒子”是由很多部分构成的。这个盒子的构成和地球结构的构成惊人地类似。由margin、border、padding、content构成。

我们的这个“内在盒子”又被分成了 4 个盒子，分别是 content box、padding box、border box 和 margin box。

给它们起了些什么名字呢？content box 写作 content-box， padding box 写作 padding-box，border box 写作 border-box。margin box没有名字。

  • 流动性丢失

当盒子设置了width之后就失去了流动性。因为盒子有了宽度就相当于是水流被两个闸限制了两侧，自然就不能流动了。

  • 与现实世界的差距

5、内联元素

[1] 理解内联元素

首先我们来理解内联元素，内敛元素并不等于display：inline的元素，内联指的是外在盒子。inline-block 和 inline-table 都是“内联元素”，因为它们的 “外在盒子”都是内联盒子。自然 display:inline 的元素也是“内联元素”。

[2] 幽灵空白节点

demo展示

<span>元素的前面还有一个宽度为 0 的空白字符，我们称为是“幽灵空白节点”。“幽灵空白节点” 实际上也是一个盒子，不过是个假想盒，名叫“strut”，中文直译为“支柱”，是一个存在于每个“行 框盒子”前面，同时具有该元素的字体和行高属性的 0 宽度的内联盒。

明白“幽灵空白节点”的存在是理解后续很多内联元素为何会这么表现的基础。

一段文本是由什么构成的呢？你会说是由字母（单个汉字）构成了单词，单词构成了段落的一行，然后多个行构成了一个段落。这就是我们要讲的几个 box 的基础。

  • inline-box

每个单词（或者段落中的行内元素）就是一个 inline-box，更严格点来讲，没有标签包裹的纯文本元素也被成为匿名 inline-box。

基线的定义就离不开本文的主角 x。 字母 x 的下边缘（线）就是我们的基线。

x-height 指的就是小写字母 x 的高度。

CSS中有些属性值的定义就和这个”x-height”的有关，最典型的代表就是：vertical-align: middle.

middle指的是基线往上1/2 “x-height”高度。我们可以近似脑补成字母x交叉点那个位置。

[3] line-height的具体值

normal

normal是默认值，它并非是由浏览器决定，而是由font-family决定。同一浏览器对于不同字体，normal有着不同的值，（1.32、1.411等）

数值

如 line-height:1.5，其最终的计算值是和当前 font-size 相乘后的值。
假设我们此时的 font-size 大小为 14px， 则 line-height 计算值是 1.5*14px=21px。

百分比

其最终的计算值是和当前 font-size 相乘后 的值。
假设我们此时的 font-size 大小为 14px，则 line-height 计算值是 150%*14px=21px。

长度值

不用计算，直接设定。

乍一看，似乎 line-height:1.5、line-height:150%和 line-height:1.5em 这 3 种 用法是一模一样的，最终的行高大小都是和font-size计算值，但是，实际上，line-height:1.5 和另外两个有一点儿不同，那就是继承细节有所差别。

设置了line-height:150%和 line-height:1.5em之后，子元素继承的并不是这些值，而是body计算后的px值。
