Javascript中常用的数组操作

发布 : 2019-04-08 分类 : Javascript 浏览 :

众所周知，在我们的日常编码里，经常会使用到数组，对于一些数据的迭代以及遍历，首先想到的是for循环。这没毛病，for循环编写语法很简单，然为遍历数组以及操作数组常用的api方法还是很多的，而且我们平时在实际中也可能经常使用。不妨总结一些常用的数组相关操作吧，方便我们以后查阅～

一、常用数组遍历方法

1、forEach

最常用的for循环是这样子的：

注意点

  • 1、forEach 可以方便的遍历数组中的每一项，提供三个参数，分别为当前值，索引，和当前数组。
  • 2、中途不能用常规操作跳出循环（不能使用break），可以用抛出异常（try/catch）的方式，但不推荐这样做。

易错点

  • 1、当数组中元素是值类型，forEach绝对不会改变数组；当是引用类型，则可以改变数组
  • 2、不支持链式操作，所以forEach之后不能再继续链式调用了

2、map

map()功能很强大，forEach()的一些局限性它很多都能解决。”map”即”映射”，也就是原数组被”映射”成对应新数组，而且对原数组毫无影响。

注意点

  • 1、新建一个数组，需要有承载对象，也意味着原始数组在调用它后不会发生变化;
  • 2、该数组中的每个元素都调用一个提供的函数后返回结果。

易错点

  • 1、map()中每个元素都要执行相应的回调函数，所以必须要有return

3、filter

Filter方法，简单滴说就是对数组进行筛选或过滤，然后返回一个新的数组，并不会对原始数组产生任何的影响。

它接受一个函数作为它的唯一参数，在数组的每个元素上调用。这个方法返回一个布尔值.

filter方法是会返回一个新数组，那我们就可以联系到了Map方法，两者结合会产生什么火花？

4、some

some()也是很好的一个方法，用于检查数组中是否有某些符合条件的元素,返回true/false;

注意点

  • 1、只要有一个满足即返回true，之后的不再执行(所以说对性能很友好！)。
  • 2、使用some之后不会影响我们的原数组

5、every

如果说some()是”||”判断，那every()就是”&&”判断，它用于检测数组中的每一项是否都满足条件，只有都满足了才会返回true。这点也很好理解：

注意点

  • 1、必须要全部满足才会返回true。如果中途碰到了一个返回false，那么就停止，直接返回false
  • 2、使用every之后不会影响我们的原数组
