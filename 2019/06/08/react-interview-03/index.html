1、状态提升（组件间通信的一种）

在 React 应用中，任何可变数据应当只有一个相对应的唯一“数据源”。通常，state 都是首先添加到需要渲染数据的组件中去。然后，如果其他组件也需要这个 state，那么你可以将它提升至这些组件的最近共同父组件中。你应当依靠自上而下的数据流，而不是尝试在不同组件间同步 state。

当需要几个组件共用状态数据的时候，可以使用状态提升技术。核心思想在于：把数据抽离到最近的共同父组件，父组件管理状态（state），然后通过属性（props）传递给子组件。如实现一个货币转换的组件，可以如下：

我们这里的跨级组件通信，指的是嵌套关系内的跨级通信。但是通常我们开发单页面应用，APP组件作为根组件，我们通常是把Context从APP往下发布，因此实现的效果为所有组件公用一个状态。

什么是Context

Context为我们提供了一个无需在每个组件之间进行props传递就可以让所有组件共享某些状态的方法。

何时用Context

Context 设计目的是为了共享那些对于一个组件树而言是“全局”的数据，例如当前认证的用户、主题或首选语言。举个例子，在下面的代码中，我们通过一个 “theme” 属性手动调整一个按钮组件的样式：

Context 主要应用场景在于很多不同层级的组件需要访问同样一些的数据，它更适合于放置全局数据。

如果你只是想避免层层传递一些属性，组件组合（component composition）有时候是一个比 context 更好的解决方案。

React.createContext

创建一个Context，当某个组件订阅了该内容时，该组件就会从组件树中找到离自己最近的那个匹配的provider中读取到当前的值。
只有当组件所处的树中没有匹配到 Provider 时，其 defaultValue 参数才会生效。这有助于在不使用 Provider 包装组件的情况下对组件进行测试。

Context.Provider

每一个Context对象都会返回一个Provider组件。它允许我们的消费者组件可以订阅Context的变化。

Provider 接收一个 value 属性，传递给消费组件。一个 Provider 可以和多个消费组件有对应关系。多个 Provider 也可以嵌套使用，里层的会覆盖外层的数据。

当 Provider 的 value 值发生变化时，它内部的所有消费组件都会重新渲染。

Class.contextType

挂在在class上的contextType属性会被赋值为一个由React.createContext创建的Context对象。这能让你使用 this.context 来消费最近 Context 上的那个值。你可以在任何生命周期中访问到它，包括 render 函数中。

PureComponent

PureComponent可以确保我们的子组件在props没有变化的时候不重新渲染。shoudComponentUpdate生命周期可以让我们去任意判断新旧props和新就state，来决定是否重新渲染，这种是最好的解决方案。PureComponent是React为我们提供的一个较为方便的方法，里面已经做了一层shouldComponentUpdate判断了。

PureComponent存在另一个问题，那就是 如果我们传入的props是一层对象，比如 name = 1; PureComponent可以完美其效果。如果我们传入的是一个对象，person: {age: 1}这种情况下，如果说我们父组件更改了person的age属性，那么不会触发Child组件的更新。所以如果使用不当容易照成不渲染的bug。

React.memo()

现在新出了一个React.memo()可以满足创建纯函数而不是一个类的需求，他是一个高阶组件。

React.memo()可接受2个参数，第一个参数为纯函数的组件，第二个参数用于对比props控制是否刷新，与shouldComponentUpdate()功能类似。

拆分的越细的组件，所传入的props就更少，更简单，PureComponent和mome出现的机会就越大。PureComponent针对class组件，memo针对函数式组件。
