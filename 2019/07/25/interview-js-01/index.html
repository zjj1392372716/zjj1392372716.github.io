一、类型

我们的 JavaScript 模块会从运行时、文法和执行过程三个阶段学习。

JavaScript 语言的每一个值都属于某一种数据类型。js中共有7种数据类型，广泛用于变量、函数参数、表达式、函数返回值等场合。

1、undefined

针对undefined我们需要知道如下：(不一定很全～)

  • 本意表示未初始化
  • typeof

无论是未初始化还是未声明，使用typeof 都是undefined.

面试题：为什么有的编程规范要求用 void 0 代替 undefined ？

JavaScript 的代码 undefined 是一个变量,而不是一个关键字。因此有些老的浏览器可以修改全局undefined值。为了防止某些情况下undefined被修改，我们推荐使用void 0来获取undefined.

在ES5中全局undefined已经只是个可读的值了，但是局部变量仍然可以修改.

认识void

void 后面跟上一个表达式，返回的都是 undefined。用 void 0 代替 undefined 能节省不少字节的大小，一些代码压缩工具就将undefined 替换为了 void 0.

总之： undefined表示未初始化，我们没必要为一个新声明的变量初始化为undefined。使用typeof操作符返回的是undefined。同时它也不是一个关键字，因此我们可以声明一个变量，他的名字就是undefined，这样的话undefined就被覆盖掉了，我们推荐使用void 0来获取undefined

2、null

Null 类型是第二个只有一个值的数据类型，这个特殊的值是 null。从逻辑角度来看，null 值表 示一个空对象指针，而这也正是使用 typeof 操作符检测 null 值时会返回”object”的原因(我们使用null表示一个空对象指针).

用途： 如果定义的变量准备在将来用于保存对象，那么最好将该变量初始化为 null 而不是其他值。

undefined实际上是派生自null的，因此相等性判断是true。

3、Boolean

任何其他类型都可以转换为Boolean类型的值。只有两种情况true、false。并且区分大小写。我们可以使用Boolean()方法来转换任意类型的值。

4、Number

  • Number 使用 IEEE754 格式来表示 整数和浮点数值，但是是基本符合，js中还有几处特殊的情况。
  • js中有+0 和 -0他们两者相等，但是仍存在一些问题.如果是加法则没问题，如果是除法我们使用了-0，结果则是负无穷大。如何区分 +0 和 -0，正是检测 1/x 是 Infinity 还是 -Infinity.
  • js数值格式

  • 根据双精度浮点数的定义，Number 类型中有效的整数范围是 -0x1fffffffffffff 至 0x1fffffffffffff，所以 Number 无法精确表示此范围外的整数。

  • 非整数的 Number 类型无法用 ==（=== 也不行） 来比较，一段著名的代码，这也正是我们第三题的问题，为什么在 JavaScript 中，0.1+0.2 不能 =0.3 （浮点运算精度问题是关键）

面试题：0.1+0.2 == 0.3? 那么浮点的比较就不能进行了吗？我们应该使用什么样的形式去比较浮点类型。

检查相差的值是否在最小精度范围内，来判断两个浮点是否相等。

  • 由于保存浮点数值需要的内存空间是保存整数值的两倍，因此 ECMAScript 会不失时机地将浮点数值 转换为整数值。

浮点数值的最高精度是 17 位小数，但在进行算术计算时其精确度远远不如整数。例如，0.1 加 0.2 的结果不是 0.3，而是 0.30000000000000004。这个小小的舍入误差会导致无法测试特定的浮点数值。
我们计算0.05+0.25得到的是0.3;

因此，永远不要测试某个特定的浮点数值。

  • 数值的范围

由于内存的限制，ECMAScript 并不能保存世界上所有的数值。ECMAScript 能够表示的最小数值保 存在 Number.MIN_VALUE 中;能够表示的最大数值保存在 Number.MAX_VALUE 中.

如果超出了这个范围那就是Infinity和-Infinity。超出范围的值是无法继续进行计算，我们可以使用isFinite()来判断一个值是否超出范围了～～

  • NaN

NaN表示不是数值，表示一个本来要返回数值的操作数未返回数值的情况。js中不同于其他语言，我们除数为0返回的是NaN，而不是报错。

NaN参与的任何运算，其结果都是NaN。NaN不等于任何，包括自身。我们可以通过isNaN()来判断一个值是否是NaN。isNaN对先对参数进行一次数值转换，对转换后的值进行判断。

6、Symbol

  • Symbol产生的原因：ES5的对象属性都是字符串，我们希望在别人的对象基础之上添加其他属性和方法，但是我们又担心会发生冲突，Symbol可以保证每一个属性都是独一无二的。
  • 产生一个Symbol,需要通过Symbol全局方法。Symbol如果接受一个对象，则会先通过toString方式去转为字符串。
  • Symbol是独一无二的
  • Symbol描述符

应用场景一：作为对象的属性

通常的对象属性都是通过字符串访问的。

使用Symbol作为属性的好处就是，我们不用为属性的冲突担心了。

Symbol 作为属性名，该属性不会出现在for…in、for…of循环中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify()返回。但是，它也不是私有属性，有一个Object.getOwnPropertySymbols方法，可以获取指定对象的所有 Symbol 属性名。

7、Object

仅仅创建 Object 的实例并没有什么用处，但关键是要理解一个重要的思想: 即在 ECMAScript 中，Object 类型是所有它的实例的基础。换句话说 Object 类型所具有的任何属性和方法也同样存在于更具体的对象中。

  • constructor: 保存着用于创建当前对象的函数
  • hasOwnProperty(propertyName): 用于检查给定的属性在当前对象实例中
  • propertyIsEnumerable(propertyName): 检测是否可以被枚举
  • toString: 返回字符串表示形式
  • valueOf: 返回对象的字符串、数值或布尔值表示。
