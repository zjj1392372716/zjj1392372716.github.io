Vue 笔记系列 (九) Vue.js 2.0 独立构建和运行时构建的区别

发布 : 2019-07-30 分类 : Vue 浏览 :

在vue2.0中，有独立构建和运行时构建两种。而在Vue1.0中并没有这么区分，只有独立构建。

官网给出了这两者的区别：

独立构建包括编译和支持 template 选项。 它也依赖于浏览器的接口的存在，所以你不能使用它来为服务器端渲染。
运行时构建不包括模板编译，不支持 template 选项。运行时构建，可以用 render 选项，但它只在单文件组件中起作用，因为单文件组件的模板是在构建时预编译到 render 函数中，运行时构建只有独立构建大小的 30%，只有 16Kb min+gzip 大小。

看的真是一脸懵～～

在 NPM 包的 dist/ 目录你将会找到很多不同的 Vue.js 构建版本。包括 完整版、运行时版本、完整版（生产环境）、运行时（生产环境）。我们会有疑惑，为什么还区分完整版和运行时版本呢？

如果需要在客户端去编译我们的模版（比如传入一个字符串给 template 选项，或挂载到一个元素上并以其 DOM 内部的 HTML 作为模板），那么我们需要加上编译器，也就是我们的完整版。

总之，独立构建的vue.js文件中含有编译器，编译器的作用就是将我们的template选项字符串进行编译和解析为render函数。

Vue.js中的编译器会将上诉的template编译为render函数

而运行时vue.js并没有template，只能使用render函数。具体写法我们需要参考官方文档。

Vue的运行过程其实分为两步：第一步：编译器将template模版编译为render函数，称为是编译过程。第二步：运行时只要执行render函数即可，称为运行过程。

Vue.js 1.0，编译器需要依赖浏览器的DOM，所以无法将编译器和运行时分开。
Vue.js 2.0，为了支持服务端渲染(server-side rendering)，编译器不能依赖浏览器的DOM，所以必须将编译器和运行时分开。

运行时构建的情况，使用的是Vue的运行时版本，不包含编译器，按道理我们只能编写render函数，我们平时的项目脚手架都是借助了vue-loader。来帮助我们编写单文件组件，而不用我们编写render了。我们此时只需要打包运行时，而不需要打包编译器。

因此。默认我们的项目是不能写template选项的，如果我们使用了，浏览器控制台或报错提示。

使用vue-cli生成项目时，会提醒使用哪种构建方式，npm包默认导出的是运行时构建，即runtime版本vue.runtime.common.js，如果想使用独立构建，需要在webpack.config.js中配置alias。
