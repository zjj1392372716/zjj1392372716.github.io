Vue 笔记系列 (六) vue 之 我们需要注意的一些细节问题

发布 : 2019-07-13 分类 : Vue 浏览 :

一、props为什么我们不能修改？

场景

vue对父组件给出的props做了修改，会在控制台抱一个错误，提示我们不应该修改，因为这样有违背单向数据流。
vue建议我们去使用传递的事件的触发来通知父组件去更新。

错误提示：Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. Instead, use a data or computed property based on the prop's value. Prop being mutated: "name"

情况一： props为基本数据类型

如果是普通类型，那么在修改时浏览器控制台会有报错信息

情况二： props为引用数据类型

如果props传入的值是引用类型，在子组件中改变其元素，不改变引用，那么不报错。

原因

vue底层通过Object.defineProperty对数据的set做了一定的处理，他有一个标识位，如果我们子组件修改props，那么标识位为false，会报错提示，如果是在父组件去更新的数据，那么标识位是true，我们可以继续 执行。

二、props的两种处理方式

在两种情况下，我们很容易忍不住想去修改 prop 中数据：

  • Prop 作为初始值传入后，子组件想把它当作局部数据来用；
  • Prop 作为原始数据传入，由子组件处理成其它数据输出。

三、$emit你可能不知道这一点

  • 可以传递回调给父组件，父组件传递消息回来

四、为什么有时候不建议我们去使用索引index作为列表的key

我们知道其实key主要用于我们组件更新时候能更快的速度去作出调整。

我们先看个例子：

点击按钮，动态添加列表长度。我们可以点击删除去删除某一个。

比如我们是[0,1,2,3]四个元素了，此时我们打算删除第三个，之后我们的数组索引为[0,1,2];

通过diff算法比较的时候，我们删除的确是最后一个。因为它认为我们的key值只是丢弃了最后一个。

这就是我们使用动态key造成的bug了～，当我们使用真实的key的时候，我们就可以避免这个发生了～
